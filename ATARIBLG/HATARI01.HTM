Hatari

Hatari is an Atari ST emulator available for MacOS, Windows, and Unix operating systems. You can download Hatari from hatari.tuxfamily.org. The latest version as of August 17, 2022 is version 1.4.1. I am using a Mac so all that follows will be oriented towards MacOS.

Configuration and Media Folder

To store Hatari configuration files and media files like floppy disk images let's create a folder and name it Hatari. Create the folder in a convenient place such as inside your home directory or on your desktop. Inside the Hatari folder let's create a cfg and a tos folder. Our Hatari folder should look like this: 

~/Hatari/
    cfg/
    tos/

TOS

Like the Atari ST computer, Hatari requires a version of The Operation System (TOS). To start with we will use emutos which is a free and modern version of TOS. You can download emutos at sourceforge.net. The latest version as of August 16, 2022 is version 1.2.1. There are several versions available but the one we want is emutos-1024k-1.2.1. Once download, unzip the file into the tos folder. This should create a sub-folder named emutos-1024k-1.2.1. Inside that folder, we will use the etos1024k.img file as our TOS.

Initial Configuration File

On my Mac Hatari likes to use ~/Library/Application Support/Hatari as the default folder for configuration files. I don't find this particularly convient, so I created an alias to the cfg folder and put it in the default folder. I then created an alias to the default folder and put it in the ~/Hatari folder. This way I can easily move between the folders when necessary

Hatari keeps its configurations in files with a .cfg prefix. We could create a configuration file by hand, but a better way is through the configuration UI built into Hatari. On my Mac, I can get to the configuration settings by pressing COMMAND-, (the usual shortcut to get to preferences). There is also a configuration UI accessible by pressing the F12 key. On the Mac the preferences are divided into ten tabs: Disks, Rom, Atari Screen, Display, Sound, System, Joysticks, Keyboard, Peripheral, and Other. Let's take a look at each tab and make changes to setup Hatari to emulate a base model Atari Mega STE with 4 megabytes of RAM.

Disks

Let's enable both Drive A and B, and make them both double sided. We can also select Auto insert B, which might come in handy later. Now let's create an fd folder in our ~/Hatari folder and set Default images location to point to that directory. We will set up hard disks at a later time.

ROM

Set TOS Image to point to the etos1024k.img file downloaded earlier. The Cartridge image setting can be left empty.

Atari Screen

Set Monitor to RGB to emulate an Atari color monitor. We will start out with a color monitor switching betweeb low and medium resolution. We can also use a Mono monitor for productivity applications when necessary, though it does require a restart.

System

Since we want to configure a basic Atari Mega STE with 4 megabytes of memory, use the following settings:

Machine Type: Mega STE

CPU: 68000 at 16 MHz

FPU: None

ST RAM Size: 4 Mib

Keyboard

For the best compatibility I change the Keyboard mapping to Scancode. With this setting the tilde (~) and the backslash (\) keys are reversed, but all other keys are mapped correctly.
